# ResultвЂ‘AutomationвЂ‘Analytics  

**ResultвЂ‘AutomationвЂ‘Analytics** is a fullвЂ‘stack web application that automates the collection, processing, and reporting of academic results for higherвЂ‘education institutions. It is built with **Node.js**, **Express**, **Sequelize** (MySQL), and a lightweight frontвЂ‘end using **EJS** and **Chart.js**.  

---  

## рџ“Њ Project Overview  

The application manages the entire lifecycle of student results:  

* **Data ingestion** вЂ“ Import raw result files (PDF/Excel) via an OCR pipeline.  
* **Data validation** вЂ“ Verify and clean imported data.  
* **Result calculation** вЂ“ Compute SGPA/CGPA, pass/fail status, and reвЂ‘valuation requests.  
* **Reporting** вЂ“ Generate dashboards, PDF reports, and analytics for departments, batches, and subjects.  

---  

## рџЋЇ Objectives  

* Provide a single source of truth for all student results.  
* Automate repetitive tasks (import, validation, calculation).  
* Offer realвЂ‘time analytics and visualizations.  
* Ensure data integrity through strict validation and audit logs.

## пїЅ Installation Guide

1. **Clone the repo**  
   ```bash
   git clone https://github.com/RaviRaushanK/result-automation-analytics.git
   cd result-automation-analytics
   ```  

2. **Install dependencies**  
   ```bash
   npm install
   ```  

 3. **Configure environment**
    * Copy `.env.example` to `.env` (or create `.env` manually).
    * Set `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `SESSION_SECRET`, etc.
    * **Important:** Do not commit your `.env` file to Git. It contains sensitive
      credentials and should be listed in `.gitignore`.
    * For development, you can use the placeholder values from `.env.example`.
    * For production, generate a strong `SESSION_SECRET` and keep it out of
      version control.

4. **Bootstrap the database**  
   ```bash
   npm run setup
   ```  
   This will:
   * Create the MySQL database (`academic_result_analytics_db`).  
   * Run migrations.  
   * Execute init scripts (`01-default-settings.js`, `02-default-admin.js`).  
   * Run seeders (`20231001000100-seed-mca.js`).  

5. **Start the server**  
   ```bash
   npm start
   ```  
   The app will be available at `http://localhost:3000`.

## рџ“љ Troubleshooting  

| Issue | Symptom | Fix |
|-------|---------|-----|
| **MySQL bootstrap fails** | `ER_ACCESS_DENIED` | Ensure `DB_USER`/`DB_PASSWORD` are correct and the user has `CREATE DATABASE` privilege. |
| **Table name mismatch** | `SequelizeDatabaseError: Table 'system_settings' doesn't exist` | Verify `tableName` in `SystemSetting` model matches migration (`system_settings`). |
| **Seeder rerun error** | `SequelizeUniqueConstraintError` | Use `--force` flag or delete existing rows before rerunning. |
| **Migration execution error** | `Error: Cannot find module 'sequelize'` | Run `npm install` before migrations. |
| **Missing database** | `ER_BAD_DB_ERROR` | Run `node scripts/bootstrap-db.js` first. |
| **Environment variable missing** | `Cannot read property 'DB_HOST' of undefined` | Ensure `.env` is present and loaded by `dotenv`. |
